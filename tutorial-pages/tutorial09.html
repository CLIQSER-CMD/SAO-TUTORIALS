Types of Cyber Attacks

Simple explanations of DDoS, MITM, SQL injection, and brute-force attacks with real-world examples and prevention strategies.

Understanding Cyber Attacks

Cyber attacks are malicious attempts to access, damage, or disrupt computer systems, networks, or data. Understanding different attack types helps you better defend against them and recognize potential threats.

DDoS (Distributed Denial of Service)

Overwhelms a system with massive amounts of traffic from multiple sources, making services unavailable to legitimate users.

Impact Level Critical

Real-World Example:

A hacker controls 10,000 infected computers (botnet) and directs them to simultaneously request pages from an e-commerce website. The site's servers can't handle the traffic and crash, preventing real customers from shopping during Black Friday.

How It Works:

  1. 1. Attacker infects thousands of devices with malware
  2. 2. Creates a botnet (network of infected devices)
  3. 3. Commands all devices to flood target with requests
  4. 4. Target's resources are exhausted
  5. 5. Legitimate users can't access the service

Prevention Strategies:

  • ‚Ä¢ Use DDoS protection services (Cloudflare, Akamai)
  • ‚Ä¢ Implement rate limiting and traffic filtering
  • ‚Ä¢ Deploy load balancers to distribute traffic
  • ‚Ä¢ Monitor network traffic for unusual patterns
  • ‚Ä¢ Have a DDoS response plan ready

Man-in-the-Middle (MITM)

Intercepts communications between two parties to eavesdrop, steal data, or manipulate information without either party knowing.

Impact Level High

Real-World Example:

You're using public WiFi at a coffee shop. An attacker on the same network intercepts your connection to your bank's website. They can see your login credentials, account balance, and even modify transaction details before sending them to the real bank.

Common MITM Scenarios:

  • ‚Ä¢ Public WiFi networks (airports, cafes, hotels)
  • ‚Ä¢ Compromised routers or network equipment
  • ‚Ä¢ Fake cell towers (IMSI catchers)
  • ‚Ä¢ DNS hijacking attacks
  • ‚Ä¢ ARP spoofing on local networks

Prevention Strategies:

  • ‚Ä¢ Use HTTPS websites (look for the lock icon)
  • ‚Ä¢ Avoid public WiFi for sensitive transactions
  • ‚Ä¢ Use VPN when on untrusted networks
  • ‚Ä¢ Verify SSL/TLS certificates
  • ‚Ä¢ Use end-to-end encrypted messaging apps
  • ‚Ä¢ Enable two-factor authentication

SQL Injection

Exploits vulnerabilities in database queries to access, modify, or delete database information through malicious SQL commands.

Impact Level High

Real-World Example:

A shopping website has a search box that directly queries the database. An attacker types: ' OR '1'='1 in the search field. This tricks the database into returning all products instead of just matching ones, potentially exposing the entire product database.

What Attackers Can Do:

  • ‚Ä¢ Read sensitive data from the database
  • ‚Ä¢ Modify or delete database content
  • ‚Ä¢ Execute administrative operations
  • ‚Ä¢ Issue commands to the operating system
  • ‚Ä¢ Access files on the database server

Prevention Strategies:

  • ‚Ä¢ Use parameterized queries/prepared statements
  • ‚Ä¢ Validate and sanitize all user inputs
  • ‚Ä¢ Limit database user permissions
  • ‚Ä¢ Use web application firewalls (WAF)
  • ‚Ä¢ Regular security testing and code reviews
  • ‚Ä¢ Keep database systems updated

Brute Force Attack

Systematically tries all possible password combinations until the correct one is found. Relies on computational power and time rather than clever techniques.

Impact Level Medium

Real-World Example:

An attacker uses automated software to try millions of password combinations against an email account. They start with common passwords like "password123", "qwerty", and dictionary words, then move to more complex combinations until they find the correct password.

Types of Brute Force:

  • Simple Brute Force: Try every possible combination
  • Dictionary Attack: Use common words and passwords
  • Hybrid Attack: Combine dictionary with variations
  • Rainbow Tables: Pre-computed password hashes

Prevention Strategies:

  • ‚Ä¢ Use strong, complex passwords (12+ characters)
  • ‚Ä¢ Implement account lockout policies
  • ‚Ä¢ Enable two-factor authentication (2FA)
  • ‚Ä¢ Use CAPTCHA after failed attempts
  • ‚Ä¢ Monitor for unusual login attempts
  • ‚Ä¢ Implement rate limiting on login forms

Other Common Attack Types

Malware Attacks

Malicious software including viruses, worms, trojans, ransomware, and spyware that infects systems to steal data, cause damage, or gain unauthorized access.

Prevention: Use antivirus software, keep systems updated, avoid suspicious downloads, be cautious with email attachments.

Phishing Attacks

Deceptive emails, messages, or websites that trick users into revealing sensitive information like passwords, credit card numbers, or personal data.

Prevention: Verify sender identities, check URLs carefully, use email filters, enable 2FA, security awareness training.

Social Engineering

Psychological manipulation to trick people into revealing confidential information or performing actions that compromise security.

Prevention: Security awareness training, verify requests through official channels, implement verification procedures.

Zero-Day Exploits

Attacks that exploit unknown vulnerabilities in software before patches are available. Highly dangerous because there are no defenses.

Prevention: Keep software updated, use intrusion detection systems, implement defense-in-depth strategies, monitor for unusual activity.

Attack Timeline: How Attacks Typically Progress

1

Reconnaissance

Attackers gather information about the target system, network, or organization.

2

Initial Access

The attacker gains initial entry through phishing, exploitation, or stolen credentials.

3

Persistence

Malware is installed or backdoors are created to maintain access.

4

Impact

The attacker achieves their goal: data theft, system damage, or financial gain.
