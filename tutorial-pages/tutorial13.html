Beginner Guide to Wireshark

Learn how to capture and analyze network traffic with Wireshark, one of the most powerful network protocol analyzers.

What is Wireshark?

Wireshark is a free and open-source packet analyzer. It is used for network troubleshooting, analysis, software and communications protocol development, and education. It allows you to see what‚Äôs happening on your network at a microscopic level.

Getting Started with Wireshark

Step 1: Download and Install

Download Wireshark from the official website: wireshark.org. Follow the installation instructions for your operating system.

Step 2: Select an Interface

Open Wireshark and you'll see a list of network interfaces. Choose the one that is currently active (e.g., "Wi-Fi" or "Ethernet").

Step 3: Start Capturing

Double-click the interface name or click the blue shark fin icon to start capturing packets.

The Wireshark Interface

The Wireshark window is divided into three main panes:

  • Packet List Pane: Displays a summary of each packet captured.
  • Packet Details Pane: Shows the protocol information for the selected packet.
  • Packet Bytes Pane: Displays the raw data of the selected packet in hexadecimal and ASCII.

Basic Filtering

A raw packet capture can be overwhelming. Use display filters to narrow down the traffic to what you're interested in.

Common Display Filters:

ip.addr == 192.168.1.1
tcp.port == 80
http
dns

You can combine filters using logical operators like && (and), || (or), and ! (not).

Example: ip.addr == 192.168.1.1 && tcp.port == 443

Practical Example: Analyzing HTTP Traffic

Let's look at a simple example of capturing and analyzing unencrypted HTTP traffic.

  1. Start a Wireshark capture.
  2. Open a web browser and navigate to a non-HTTPS website (e.g., http://info.cern.ch).
  3. Stop the capture in Wireshark.
  4. In the filter bar, type http and press Enter.
  5. You will see the HTTP GET request from your browser to the server, and the server's response.
  6. Select a packet and expand the "Hypertext Transfer Protocol" section in the Packet Details pane to see the request/response headers and data.
